SIMD.fromFloat32x4

SIMD.js has been taken out of active development in TC39 and removed from Stage 3. It is not being pursued by web browsers for implementation anymore. SIMD operations exposed to the web are under active development within WebAssembly, with operations based on the SIMD.js operations.

The static SIMD.%type%.fromFloat32x4() method creates a new SIMD data type with a float conversion from a Float32x4.

Syntax

SIMD.Int32x4.fromFloat32x4(t)
SIMD.Uint32x4.fromFloat32x4(t)

Parameters

t
A Float32x4 SIMD type to convert from.

Return value

A new SIMD data type with a float conversion from a Float32x4.

Exceptions

  • A RangeError, if the conversion from floating-point to integer loses precision (if the lane values are smaller than -2147483649.0 or larger than 2147483648.0)

Examples

SIMD.Int32x4.fromFloat32x4

A floating-point to integer conversion of t is performed.

var t = SIMD.Float32x4(1.0, 2.0, 3.0, 4.0);
SIMD.Int32x4.fromFloat32x4(t);
// Int32x4[1, 2, 3, 4]

var t = SIMD.Float32x4(1.0, 2.0, 3.0, 2147483650.0);
SIMD.Int32x4.fromFloat32x4(t);
// RangeError: SIMD conversion loses precision
